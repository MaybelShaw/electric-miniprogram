# 小程序首页入口结构校正计划

## 目标

把小程序用户侧入口补充为“品牌专区”和“活动专区”两个运营入口，同时保留首页原有商品导购模块。首页保留搜索、轮播、礼品/设计师/爆品专区、品类专区、品牌专区和全部商品，用户既可以通过运营入口进入品牌或活动，也可以继续在首页直接浏览商品。

## 范围

- 首页新增 `品牌专区`、`活动专区` 两个并排入口，但不移除原有商品导购模块。
- 品牌专区复用合作方店铺列表，点击合作方后直接进入该店铺商品列表。
- 活动专区复用动态专区数据，先展示活动入口列表，点击具体活动后进入活动详情和活动商品列表。
- 首页恢复礼品/设计师/爆品专区、品类专区、品牌专区和全部商品列表。
- 不改后端接口、订单、购物车、支付、商户后台权限和 Django 后台。

## 执行步骤

1. 调整 `frontend/src/pages/home/index.tsx` 和样式，保留搜索、轮播和两个主入口，并恢复原有首页商品导购模块。
2. 调整 `frontend/src/pages/store-list/index.tsx`，页面标题改为品牌专区，合作方卡片点击直达商品列表。
3. 调整 `frontend/src/pages/special-zone/index.tsx`，增加 `mode=activity-list` 活动列表模式，并在详情模式展示活动标题和副标题。
4. 运行小程序构建验证：`cd frontend && npm run build:weapp`。

## 验收标准

- 首页首屏能看到两个并排入口：`品牌专区`、`活动专区`。
- 首页能看到 `礼品专区`、`品类专区`、`品牌专区` 和 `全部商品` 模块。
- 点击品牌专区进入品牌/合作方列表。
- 点击某个合作方后直接进入该合作方商品列表，不进入店铺主页。
- 点击活动专区进入活动列表。
- 点击某个活动进入活动详情，并能看到活动商品。
- `npm run build:weapp` 通过。

## 默认假设

- 首页搜索栏和首页轮播继续保留。
- 活动详情先复用 `special-zone` 页面，不新建独立活动详情页。
- 活动入口暂用 `SpecialZone.kind` 的 `activity` 和 `promotion` 作为前端筛选依据。
