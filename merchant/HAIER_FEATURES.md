# 商户后台 - 海尔功能快速参考

## 🎯 核心功能

### 1. 海尔订单识别
- ✅ 自动识别海尔产品订单
- ✅ 显示"海尔订单"标签
- ✅ 显示推送状态

### 2. 手动推送订单
- ✅ 已支付订单可推送
- ✅ 配置订单来源和店铺
- ✅ 推送成功自动更新状态

### 3. 查询物流信息
- ✅ 已推送订单可查询
- ✅ 显示多种物流信息
- ✅ 实时查询最新状态

## 📋 操作流程

### 推送海尔订单

```
1. 进入订单管理页面
   ↓
2. 找到"已支付"的海尔订单
   ↓
3. 点击"推送海尔"按钮
   ↓
4. 填写订单来源和店铺名称
   ↓
5. 确认订单信息
   ↓
6. 点击"确定"推送
   ↓
7. 等待推送结果
   ↓
8. 推送成功，状态更新为"已推送"
```

### 查询物流信息

```
1. 找到已推送的海尔订单
   ↓
2. 点击"查询物流"按钮
   ↓
3. 查看物流信息
   ↓
4. 关闭对话框
```

## 🔍 界面元素

### 订单列表

| 列名 | 说明 |
|------|------|
| 订单号 | 可复制 |
| 用户名 | 下单用户 |
| 商品名称 | 商品信息 |
| 数量 | 购买数量 |
| 总金额 | 订单金额 |
| 状态 | 订单状态 |
| **海尔订单** | 是/否，已推送/未推送 |
| 操作 | 查看、推送海尔、查询物流等 |

### 操作按钮

| 按钮 | 显示条件 | 功能 |
|------|---------|------|
| 查看 | 所有订单 | 查看订单详情 |
| **推送海尔** | 已支付且未推送的海尔订单 | 推送到海尔系统 |
| **查询物流** | 已推送的海尔订单 | 查询物流信息 |
| 发货 | 已支付订单 | 标记为已发货 |
| 完成 | 已发货订单 | 标记为已完成 |
| 取消 | 待支付/已支付订单 | 取消订单 |

## 💡 使用技巧

### 1. 批量推送

如果有多个海尔订单需要推送：
1. 逐个点击"推送海尔"按钮
2. 使用相同的配置参数
3. 推送完成后刷新列表

### 2. 推送前检查

推送前确认：
- ✅ 订单信息完整
- ✅ 收货地址正确
- ✅ 联系方式有效
- ✅ 商品编码正确

### 3. 物流跟踪

定期查询物流信息：
1. 推送后等待海尔系统处理
2. 定期点击"查询物流"
3. 关注物流状态变化

### 4. 错误处理

如果推送失败：
1. 查看错误提示
2. 检查订单信息
3. 稍后重试
4. 联系技术支持

## 🎨 视觉标识

### 海尔订单标签

- **蓝色"是"标签** - 海尔产品订单
- **灰色"否"标签** - 普通商品订单

### 推送状态标签

- **橙色"未推送"** - 待推送到海尔
- **绿色"已推送"** - 已推送到海尔

### 订单状态标签

- **橙色"待支付"** - 等待用户支付
- **蓝色"已支付"** - 已支付，可推送
- **青色"已发货"** - 已发货
- **绿色"已完成"** - 订单完成
- **红色"已取消"** - 订单取消

## ⚠️ 注意事项

### 1. 推送时机

- 建议在订单支付成功后立即推送
- 推送前确认订单信息无误
- 推送后无法撤销

### 2. 数据准确性

- 确保收货地址包含省市区县信息
- 确保联系电话格式正确
- 确保商品编码正确

### 3. 系统配置

推送功能需要后端配置：
- 海尔OAuth2.0认证信息
- 易理货系统认证信息
- 客户编码和密码

### 4. 权限要求

- 需要管理员权限
- 普通用户无法推送订单
